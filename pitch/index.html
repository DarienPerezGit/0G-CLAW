The problem

Personal AI agents are stuck on local disk and centralized inference.

Lose the laptop, lose the agent. Trust the model? Trust the API key. Two failure modes any "AI assistant" inherits the moment it goes to production.

OpenClaw today

Local-first, single-tenant, opaque

  • Memory lives in ~/.openclaw/agents/<id>/sessions/*.jsonl. Disk dies, agent dies.
  • Inference goes to OpenAI / Anthropic. Censorable, opaque, vendor-locked.
  • Agent identity is the laptop. Nothing portable, nothing addressable.
0G-Claw

Decentralized, portable, verifiable

  • Memory persists in 0G Storage (KV for state, Log for history).
  • Inference runs on 0G Compute with TeeML proofs on every call.
  • Same wallet, any machine, same agent — identity is portable, not local.
How it works

Two interfaces. Many adapters. Zero coupling.

Memory and compute are TypeScript interfaces. IMemoryAdapter and IComputeAdapter are the only contracts the agent sees. Implementations are swappable: 0G in production, local for CI, anything else for your own deployment.

Capabilities

Four properties you can't get from a local-first agent.

Each capability shows up in code, not slogans. The framework's only job is making sure these are the cheap default — adapters do the heavy lifting.

CAPABILITY 01
Verifiable inference

Every chat() from 0GComputeAdapter returns a TeeML proof. The agent surfaces it per turn — no model can be silently swapped behind a proxy.

const result = await compute.chat(messages);
// → result.content
// → result.verificationHash: "0xabc…"
//   ↑ provider-signed TeeML proof
CAPABILITY 02
Shared memory across agents

The agent ID is derived from the topic, not the laptop. Two processes — same wallet, same topic — read and write the same 0G namespace. No central orchestrator.

const agentId = `research-${sha256(topic).slice(0, 8)}`;
// → research-7f3a2c19
// Same on any machine with the same wallet.
CAPABILITY 03
Replayable execution

Every state mutation goes to an append-only Log Store. The full report can be reconstructed from the log alone, even if the snapshot is lost.

const history = await memory.loadHistory(agentId, sessionId);
// → SessionMessage[] in causal order
// Every Finding, every retry, fully ordered.
CAPABILITY 04
Portable identity

The wallet is the identity. Run from any machine, same agent appears. ENS integration (planned) makes it addressable by name.

$ MEMORY_ADAPTER=0g pnpm example:research
$ # laptop B, same wallet:
$ MEMORY_ADAPTER=0g pnpm example:research
  → "Found 4 prior findings — skipping"
Reference agents

Two agent shapes on the same adapters.

A framework is only a framework if it supports more than one agent. We ship two — a chat loop and a research pipeline — built on the same IMemoryAdapter + IComputeAdapter contracts. Zero coupling between them.

basic-agent
CHAT LOOP

Multi-turn conversation. Loads prior session on startup, appends each turn to KV + Log. Detects TTY for interactive vs scripted modes (Docker / CI runs the latter).

$SESSION_ID=… MEMORY_ADAPTER=0g pnpm example:basic
  • Context window of last 20 messages, bounded
  • Same code path against any backend (0G or local stub)
  • Resumes by SESSION_ID on any machine with the same wallet
  • Prints verificationHash per turn when on 0G Compute
research-agent
PLAN → RESEARCH → SYNTHESIZE

Topic-driven pipeline. LLM plans sub-questions, queries Wikipedia per sub-question, persists each finding, synthesizes a cited report. Resumable from any machine.

$RESEARCH_TOPIC="0G" MEMORY_ADAPTER=0g pnpm example:research
  • Tool abstraction (ITool) — Wikipedia + memory recall
  • Durability: log-store-first; KV snapshot follows
  • example:research:inspect — read-only memory viewer
  • Each finding carries its own verificationHash
Same adapter contract. Completely different agent shape. That's the framework story — judges can verify the diff at git diff main -- adapters/: nothing changed.

Try it

30 seconds from clone to verifiable research.

Real Wikipedia API. Real LLM. Real on-chain attestation per finding. No build step, no Docker required (Docker is available — see README), and falls back to local adapters cleanly if you don't have 0G credentials yet.

01
Clone and install
$git clone https://github.com/DarienPerezGit/0G-CLAW.git
$cd 0G-CLAW # 1610 lines of TS, 79 passing tests
$pnpm install
02
Configure (optional — skip for local fallback)
$cp .env.example .env
$# Fill in OG_PRIVATE_KEY + OG_COMPUTE_PROVIDER for full 0G mode.
$# See README "broker funding requirements" for the one-time setup.

Without these, the agent runs against LocalMemoryAdapter + a stub compute adapter — pipeline structure stays exercised end-to-end.

03
Run the chat agent
$MEMORY_ADAPTER=0g pnpm example:basic # 0G memory + local stub
$MEMORY_ADAPTER=0g COMPUTE_ADAPTER=0g pnpm example:basic # full
04
Run the research agent
$RESEARCH_TOPIC="0G Network architecture" \
    MEMORY_ADAPTER=0g COMPUTE_ADAPTER=0g \
    pnpm example:research

$# Then on any other machine with the same wallet:
$RESEARCH_TOPIC="0G Network architecture" \
    MEMORY_ADAPTER=0g pnpm example:research:inspect

Same wallet, same topic, any machine = same research. The headline of the framework story.
