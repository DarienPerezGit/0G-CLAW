02 · How it composes

Two interfaces. Many agents. Zero coupling.

The adapter pattern is the contract. OpenClaw doesn't know which memory or compute backend is wired. Two example agents prove the contract is real — both use the same primitives, in completely different shapes.

01
Verifiable inference
Every chat() from 0GComputeAdapter returns a verificationHash. Surfaced per turn in basic-agent and per finding in research-agent.
02
Shared memory
Same wallet + same topic = same memory namespace, anywhere. The research-agent derives agentId from sha256(topic).
03
Replayable execution
The Log Store is append-only. The full report can be reconstructed from the log alone via loadHistory.
04
Portable identity
The wallet is the identity. Run from any machine — same agent. ENS will make this addressable by name (track 2).

05 · Demo strategy

Three minutes, five acts, zero panic.

The demo is structured so any single failure during the live run is recoverable without re-doing setup. Acts 2 and 3 are the prize-winning beats; act 4 is the safety net; acts 1 and 5 are framing.

Act 01
0:00 — 0:30
Hook
"OpenClaw is great — but its memory dies with the laptop and inference goes to OpenAI. 0G-Claw fixes both."
Act 02 · Headline
0:30 — 1:45
Portable memory live
Run research-agent on machine A. Kill it. Resume from machine B with the same wallet — agent picks up exactly where A left off, from 0G Storage.
Act 03
1:45 — 2:30
Verifiable inference
Show 21/21 compute integration tests passing, point at verificationHash. Optionally a live COMPUTE_ADAPTER=0g call if testnet looks healthy.
Act 04
2:30 — 2:50
Fallback resilience
Switch to MEMORY_ADAPTER=local COMPUTE_ADAPTER=local. Same agent, same code path, no creds. The contract is the interface.
Act 05
2:50 — 3:00
Close
"Two adapters, four 0G-native capabilities, two reference agents — OpenClaw didn't change. We just made the persistence layer decentralized." Repo + license shot.
⚠ Failure modes during the live demo
0G Storage upload stalls in act 2
Drop to MEMORY_ADAPTER=local. Narration: "for time, switching to local — same interface, same code path."
Compute call hangs in act 3
Skip the live call, lean on the 21/21 test artifact already on screen. The proof is already pre-recorded.
Wallet runs out of OG mid-demo
Use a pre-recorded session from the demo-data branch — resume reads from history, no fresh writes needed.
Audience asks "is this really verifiable?"
Run pnpm example:research:inspect on the spot — every finding's verificationHash prints in the table.
